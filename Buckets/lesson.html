DIV and SPAN

There are two elements which have no semantic meaning and they exist solely to allow for presentation hooks. These two elements are DIV and SPAN. The only difference between the two is that DIV is a "block" level element and SPAN is a "inline" level element. We will explain this distinction shortly.

CSS

Where HTML sets down the Structure of your content, CSS sets down the Style of your Content. CSS is a completely different language compared to HTML and it has its own syntax and rules. However, together, HTML and CSS can create almost any visual style you can think of (see CSS Zen Garden for examples). Broadly speaking, you can break up CSS into three fundamental parts: selectors, properties, and values.

  1. Selectors define the "path" for getting at a particular element (or sets of elements) in the HTML document. See CSS Selectors for the full list.
  2. Properties are the list of available attributes of elements that can be changed (e.g. colour, border, font size, background, margins, etc.). Not all properties apply to all elements, but there are a universal set that apply to all. See CSS Properties for the full list.
  3. Values define the value of the attribute you want to set. Each property has a distinct set of valud values. Look at individual Properties (from the above link) to see what their valid values are.

What can't it do?!

While HTML and CSS are really powerful and have the ability to create varied visual styles, they can only provide limited amounts of interactivity to a web site. HTML by itself is completely static and has no mechanism for interactivity. CSS has some selectors and properties that react to user inputs like click, hover, etc, but they are comparatively limited. To achieve full interactivity, you need the third pillar of web development: JavaScript.
